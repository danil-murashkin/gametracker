# Препараты и перки старой системы (FT3 / PDA)

Официальное описание NFC-препаратов и перков персонажа для переноса логики в новую систему.

| Источник | Содержание |
|----------|------------|
| `Препараты.xlsx` | Каталог тегов (кол. A — имя в списке, кол. B — NFC-payload) |
| `pda_8266_b8-1.ino`, `ml-ev5.h` | Исполнение на PDA |
| `FT3_doctor_drugs.sc` | Препараты доктора (`PERK:DOC`) — полный каталог §2.1 |
| `FT3_speeches_converted.sc` | Речи оратора (`PERK:ORA`) — полный каталог §2.2 |
| `FT3_armors.sc` | Теги брони — каталог §2.3 |

Имена в колонке A и поле `NAME` на теге могут отличаться. Ошибки Excel поправлены в описании (пометки «Исправление»).

## 1. Модель эффектов

### 1.1. Время

- Игровой цикл `MLCount` раз в **1 с** (`MLTimeout = 1000`).
- `REP`, `AFT`, `INT` задаются в **секундах (тиках ML)**.
- Пример: `REP300` ≈ 5 мин, `REP900` ≈ 15 мин, `REP3600` ≈ 1 ч, `REP7200` ≈ 2 ч, `REP10800` ≈ 3 ч, `REP14400` ≈ 4 ч.

### 1.2. Параметры `TAR*` (индексы MLP)

| TAR | Константа | Смысл |
|-----|-----------|--------|
| 1 | `STRENGTH` | Сила (база) |
| 2 | `PERCEPTION` | Восприятие (база) |
| 3 | `ENDURANCE` | Выносливость (база) |
| 4 | `CHARISMA` | Харизма (база) |
| 5 | `INTELLECT` | Интеллект (база) |
| 6 | `AGILITY` | Ловкость (база) |
| 7 | `LUCK` | Удача (база) |
| 8 | `MODSTRENGTH` | Модификатор силы |
| 9 | `MODPERCEPTION` | Модификатор восприятия |
| 10 | `MODENDURANCE` | Модификатор выносливости |
| 11 | `MODCHARISMA` | Модификатор харизмы |
| 12 | `MODINTELLECT` | Модификатор интеллекта |
| 13 | `MODAGILITY` | Модификатор ловкости |
| 14 | `MODLUCK` | Модификатор удачи |
| 15 | `LIFE` | Текущая жизнь |
| 16 | `MAXLIFE` | Макс. жизнь |
| 17 | `FOOD` | Запас еды |
| 18 | `WATER` | Запас воды |
| 19 | `FOODPERCYC` | Расход еды / тик (типично 3) |
| 20 | `WATERPERCYC` | Расход воды / тик (типично 5) |
| 21 | `RADRESIST` | Сопротивление радиации |
| 22 | `CHEMBIORESIST` | Сопротивление хим/био |
| 23 | `PHYSICALRESIST` | Сопротивление физ. урону |
| 24 | `RADDOSE` | Накопленная доза радиации |
| 25 | `RADNOW` | Острая радиация текущего тика |
| 26 | `CHEMBIO` | Хим/био урон тика |
| 27 | `PHYSICAL` | Физ. урон тика |
| 28 | `PERK` | Перк: 0 нет, 1 доктор, 2 выживальщик (SUR), 3 король вечеринок, 4 оратор, 5 инженер |
| 29 | `PLAYERID` | ID игрока |
| 30 | `STARTFOOD` | Стартовая еда после оживления |
| 31 | `STARTWATER` | Стартовая вода после оживления |
| 32 | `RACE` | Раса |
| 33 | `AMBIENTTEMP` | Температура среды |
| 34 | `MODTEMP` | Модификатор температуры |
| 35 | `PLAYERTEMP` | Температура игрока |
| 36 | `SHELTER` | Укрытие |
| 37 | `ARMORSTATE` | Состояние брони |
| 38 | `MAXARMOR` | Максимум брони |
| 39 | `ARADRESIST` | Броня: сопр. радиации |
| 40 | `APHYSRESIST` | Броня: сопр. физ. |
| 41 | `ACHEMBIORESIST` | Броня: сопр. хим/био |
| 42 | `ATHERMOINSULATION` | Броня: термоизоляция |

**Важно:** каждый тик `MLCount` обнуляет `MLP[8..14]` (модификаторы SPECIAL) и `MLP[21..23]` (сопротивления), затем вызывает `CalcEffects()`. Поэтому временные баффы на эти поля пишутся как `ADDx REPn` и «держатся», пока эффект жив.

Еда/вода: время запаса ≈ `FOOD/FOODPERCYC` и `WATER/WATERPERCYC` секунд.

### 1.3. DSL эффекта (одна строка = один `EffectEntry`)

Разделитель строк эффектов — LF (`outEffDiv`). Токены внутри строки — пробел (`inEffDiv`).

| Токен | Назначение |
|-------|------------|
| `TARn` | Целевой индекс MLP |
| `ADDx` / `SETx` | Прибавить / присвоить; `x` может быть выражением (`v16/2`, `fQTY…`) через `Interprete` |
| `REPn` | Сколько раз сработать (по умолчанию 1). `< -9` = бесконечно |
| `INTn` | Пауза между срабатываниями (после тика `After = Interval`) |
| `AFTn` | Отложить первое срабатывание на n тиков |
| `EIDc` | Identifier эффекта (1 символ) |
| `REMc` | При срабатывании деактивировать один эффект с Identifier `c` |
| `CONexpr` | Если `Interprete(expr)<=0` — эффект **не добавляется** |

Выражения: `vN` = `MLP[N]`; `fQTYc` = число эффектов с ID `c`; `fDELc` = то же + удаление; `fRNDn` = `random(n)` (0…n−1); операторы `+ - * / > < = != & |`.

### 1.4. Зависимости (общий паттерн)

Химия с зависимостью обычно:
1. Вешает короткий бафф на MOD/resist.
2. Если зависимость уже есть — `REM` старых маркеров (с временным `EID d`) и **переназначение** дебаффа через `CON fQTYd>0`.
3. Если зависимости ещё нет — ролл нового маркера: условие `v28!=3` (не король вечеринок) и часто проверка удачи `v7+v14 < fRNDn`. У короля вечеринок новые зависимости **не прилипают**.
4. Дебафф с `AFT` (отсрочка) и длинным `REP` (обычно 7200 = 2 ч).

Полное снятие маркеров: **Аддиктол** или препараты доктора (`PERK:DOC` → Splash `FT3eREMx`). Маркеры: `b e v u m n t r s k q`.

| Маркер | Источник (в этом Excel) | Типичный дебафф |
|--------|-------------------------|-----------------|
| `b` / `e` | Баффаут | сила −1 / выносливость −1 |
| `v` / `u` | Винт, Супервинт | ловкость −1 / интеллект −2 |
| `m` / `n` | Мед-Х | химсопротивление −10 / восприятие −1 |
| `t` | Ментаты | интеллект −2 |
| `r` / `s` | Рад-Х | сила −1 / выносливость −1 |
| `q` | Квантовая Ядер-Кола | сила −2 |
| `k` | (нет в Excel) | снимается Аддиктолом |

### 1.5. Ключи NFC-тега

| Ключ | Смысл |
|------|--------|
| `PROTO` | Должен быть `FT3` для PDA |
| `NAME` / `TEXT` / `USES` | Отображение и счётчик |
| `EFFECTS` | Блок DSL |
| `TYPE` | `HOLO`, `CANN`, `CANW`, `ANA`, `EMP`, … |
| `PERK` | Метка тега под перк: `SUR` / `DOC` / `ORA` / `ENG` (см. §6) |
| `ONLY` | Условие через `Interprete` (например `v28=5`) |
| `SPLASH` / `SPLASHPOW` | Wi‑Fi splash после успешного consume |
| `REVIVE` | Минуты до оживления; форсирует смерть |
| `RESET` | `SOFT` / `HARD` |
| `PERKNOW` | Сдвиг таймера перка |

Обычные препараты работают только при `LIFE > 0` (кроме `REVIVE`, `RESET`, каннибальских ножей на трупе).

## 2. Каталог препаратов

Теги из `Препараты.xlsx`. Дополнительные наборы: §2.1 доктор, §2.2 речи, §2.3 броня.

### RadAway

- **Категория:** Химия / антирадин
- **USES:** 1
- **TEXT:** Выводит 500 ед. радиации
- **Действие:**
  - Разово уменьшает накопленную дозу радиации `RADDOSE` (TAR24) на 500.
- **Технические заметки:**
  - Одноразовый `ADD` (не модификатор). Отрицательные значения затем клампятся в `MLCount`.
- **EFFECTS (сырой DSL):**
  ```
  TAR24 ADD-500
  ```

### Аддиктол

- **Категория:** Химия / лечение зависимости
- **USES:** 1
- **TEXT:** Снимает все зависимости
- **Действие:**
  - Снимает эффекты-зависимости по идентификаторам: `b e v u m n t r s k q` (по одному на каждый `REM*`).
- **Технические заметки:**
  - Каждая строка `REMx` — отдельный эффект без TAR: при срабатывании обнуляет `Repeats` у **одного** эффекта с `Identifier==x`.
  - Буква `k` снимается Аддиктолом, но в этом Excel ни один препарат её не выставляет.
- **EFFECTS (сырой DSL):**
  ```
  REMb
  REMe
  REMv
  REMu
  REMm
  REMn
  REMt
  REMr
  REMs
  REMk
  REMq
  ```

### Баффаут

- **Категория:** Химия (с зависимостью)
- **USES:** 1
- **TEXT:** Сила и Выносливость +2 на 5 минут
- **Действие:**
  - `MODSTRENGTH` +2 и `MODENDURANCE` +2 на 5 мин (`TAR8/10 ADD2 REP300`).
  - Модификаторы каждый тик обнуляются и снова накладываются эффектом → стабильный бафф на время REP.
- **Побочки / зависимость:**
  - **Зависимость (Баффаут):**
  - Маркеры эффектов: основной `b`, вторичный `e`.
  - При повторном приёме, если маркер уже есть: старый эффект снимается (`REMb` / `REMe` с `EID d`), затем зависимость **переназначается** (`CON fQTYd>0`).
  - Новый ролл: не «король вечеринок» (`v28!=3`), нет активного `b`, и `Удача+ModУдача < random(17)` (диапазон 0…16; выше удача — реже зависимость).
  - Отсрочка до дебаффа: **15 мин** (`AFT900`); длительность: **2 ч** (`REP7200`).
  - Эффект зависимости: сила −1 (`TAR8 ADD-1`, EID `b`); выносливость −1 (`TAR10 ADD-1`, EID `e`, ставится если уже есть `b`)
  - Перк «король вечеринок» (`PERK=3`): новые зависимости **не прилипают** (`v28!=3`); уже висящую не снимает (см. §6.3).
- **EFFECTS (сырой DSL):**
  ```
  TAR8 ADD2 REP300
  TAR10 ADD2 REP300
  REMb EIDd CONfQTYb>0
  REMe EIDd CONfQTYe>0
  TAR8 ADD-1 REP7200 AFT900 EIDb CONfQTYd>0
  TAR8 ADD-1 REP7200 AFT900 EIDb CONv28!=3&fQTYb=0&v7+v14<fRND17
  TAR10 ADD-1 REP7200 AFT900 EIDe CONfQTYb>0
  ```

### Винт

- **Категория:** Химия (с зависимостью)
- **USES:** 1
- **TEXT:** Ловкость +3 на 15 минут. От винта!
- **Действие:**
  - `MODAGILITY` +3 на 15 мин.
- **Побочки / зависимость:**
  - **Зависимость (Винт):**
  - Маркеры эффектов: основной `v`, вторичный `u`.
  - При повторном приёме, если маркер уже есть: старый эффект снимается (`REMv` / `REMu` с `EID d`), затем зависимость **переназначается** (`CON fQTYd>0`).
  - Новый ролл: если не «король вечеринок» (`v28!=3`) и нет `v`, зависимость **ставится всегда** (без проверки удачи).
  - Отсрочка до дебаффа: **30 мин** (`AFT1800`); длительность: **2 ч** (`REP7200`).
  - Эффект зависимости: ловкость −1 (`TAR13`, EID `v`); интеллект −2 (`TAR12`, EID `u`, если есть `v`)
  - Перк «король вечеринок» (`PERK=3`): новые зависимости **не прилипают** (`v28!=3`); уже висящую не снимает (см. §6.3).
- **Технические заметки:**
  - В отличие от Баффаута/Ментатов/Мед-Х/Рад-Х, у Винта **нет** проверки удачи на новый ролл.
- **EFFECTS (сырой DSL):**
  ```
  TAR13 ADD3 REP900
  REMv EIDd CONfQTYv>0
  REMu EIDd CONfQTYu>0
  TAR13 ADD-1 REP7200 AFT1800 EIDv CONfQTYd>0
  TAR13 ADD-1 REP7200 AFT1800 EIDv CONv28!=3&fQTYv=0
  TAR12 ADD-2 REP7200 AFT1800 EIDu CONfQTYv>0
  ```

## 2.3. Броня (`FT3_armors.sc`)

Одноразовые теги экипировки (`USES:1`): при Consume выставляют параметры брони `TAR37–42` (и опционально расход воды). Поля `ARADRESIST` / `APHYSRESIST` / `ACHEMBIORESIST` / `ATHERMOINSULATION` / `MAXARMOR` / `ARMORSTATE` **не** обнуляются каждый тик (в отличие от личных resist 21–23) — `SET` закрепляется.

Эффективная защита в бою: `map(ARMORSTATE, 0…MAXARMOR → 0…A*RESIST)` в `MLCount`.

### Тестовая броня

- **Имя в источнике:** Тестовая броня

- **USES:** 1
- **TEXT:** Тестовая броня
- **Источник:** `FT3_armors.sc`
- **Действие:**
  - Экипировка брони: `ARADRESIST`=5, `APHYSRESIST`=5, `ACHEMBIORESIST`=5, `ATHERMOINSULATION`=2; прочность 200/200 (`ARMORSTATE`/`MAXARMOR`).
- **Технические заметки:**
  - В `.sc` у EFFECTS есть комментарии после таба (`#…`); при записи на NFC их лучше отрезать.
  - Повторный Consume другой брони перезапишет те же TAR* новыми SET.
- **EFFECTS (сырой DSL):**
  ```
  TAR39 SET5
  TAR40 SET5
  TAR41 SET5
  TAR42 SET2
  TAR38 SET200
  TAR37 SET200
  ```

## 6. Перки персонажа

Значение `MLP[PERK]` (`TAR28`). Откат перка (кроме короля вечеринок):  
`PerkTimeoutMin = (11 − AGILITY + ModAGILITY) × 10` минут.  
Теги с `PERK:DOC` / `ORA` / `ENG` **нельзя** принять обычным Consume — только через действие перка (`PerkTag`). Теги `PERK:SUR` можно и есть/пить как обычно, и очищать перком.

### 6.1. Доктор (`PERK=1`, теги `PERK:DOC`)

**Смысл:** восстанавливает «химию» — раздаёт медицинские препараты окружающим через Wi‑Fi Splash (UI: «раздать препарат»).

**Механика (`PerkTag`):**
1. Нужен тег с `PERK:DOC` и `USES > 0`; USES уменьшается на 1.
2. Успех, если `Восприятие + Интеллект + Ловкость` (с модификаторами) **≥ 21**.
3. Длительность splash: `map(stats, 21…30 → 4…10) × 60` секунд, дописывается как `… REP{sec}` к `SPLASH`.
4. Мощность: `SPLASHPOW` (часто 10).

Полный список DOC-тегов: **§2.1**. Префикс splash `FT3e…` на PDA → `AddEffect(...)` у принявших в зоне.

### 6.2. Выживальщик / Survivalist (`PERK=2`, теги `PERK:SUR`)

**Смысл:** может очищать еду и воду от радиации (UI: «очистить еду/воду»).

**Механика (`PerkTag`):**
1. Нужен тег с `PERK:SUR` и `USES > 0`.
2. После очистки: `USES = min(USES, ENDURANCE + ModENDURANCE − 6)` (при результате &lt; 0 → 0).
3. В `EFFECTS` дописывается `TAR25 SET0` (`RADNOW = 0`).
4. `TEXT` заменяется на «Очищено от радиации».
5. Сами эффекты еды/воды при последующем Consume сохраняются, острая радиация с тега уже снята.

В каталоге §2 такие позиции помечены ключом `PERK:SUR` (грязная вода, молоко, мутафрукт, колы, снеки и т.д.).

### 6.3. Король вечеринок (`PERK=3`)

**Смысл:** зависимости к нему не прилипают — игнорирует ролл новых зависимостей от химии.

**Механика:**
- Отдельного `PerkTag` / таймера перка нет (UI только показывает статус).
- Во всех схемах зависимости условие нового ролла содержит `v28!=3`. При `PERK=3` ветка не срабатывает → маркеры `b/v/m/t/r/q/…` **не назначаются**.
- Уже висящую зависимость этот перк сам не снимает (нужны Аддиктол или доктор).
- Повторный приём при уже существующем маркере по-прежнему может **переназначить** дебафф через `REM` + `CON fQTYd>0` — иммунитет именно к *новому* «прилипанию».

### 6.4. Оратор (`PERK=4`, теги `PERK:ORA`)

**Смысл:** использует речи для формирования воздействия на окружающих через Splash.

**Механика (`PerkTag`):**
1. Нужен тег-речь с `PERK:ORA` и `USES > 0`; USES −1.
2. Успех, если `Харизма + ModХаризма > 6`.
3. Длительность: `(CHA + ModCHA − 6) × 600` секунд → суффикс ` REP{sec}` к `SPLASH`.
4. Мощность: `SPLASHPOW` (в речах часто 17).

Полный список речей: **§2.2**. Splash `FT3e…` → `AddEffect` на слушателей в зоне.

### 6.5. Инженер (`PERK=5`, теги `PERK:ENG` и ремонтные наборы)

**Смысл:** временно отключает аномалии применением инструмента; также чинит/собирает броню ремонтными тегами.

**Инструмент (`PerkTag` + `PERK:ENG`, в каталоге — «Железяка»):**
1. USES −1 при успехе.
2. Успех, если `INTELLECT > 6` (без модификатора в формуле).
3. Splash SSID: `{SPLASH|по умолчанию FT3}` + `K` + `{(INTELLECT − 6) × 3}` — команда отключения аномалий на время, пропорциональное интеллекту.
4. Мощность: `SPLASHPOW` (у инструмента 15).

**Ремонт / силовая броня** (обычный Consume при `ONLY:v28=5`):
- «Ремонтный комплект» → splash `FT3eTAR37 SETv38` (полная починка `ARMORSTATE`).
- «Запчасти брони» → `REPDEV-A`.
- «Комплект деталей» → `RESDEV-C` (восстановление силовой брони).
