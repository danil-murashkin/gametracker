# GameTracker

---

## Введение

В страйкболе **сценарные игры** требуют не только **экипировки**, но и **электроники** для механик и сюжета. Она связывает **роль**, **действия игрока** и **отклик мира** — квесты, зоны, предметы, реакция организаторов. Цель проекта — **система трекеров** и **полевого оборудования**, работающих в одной **беспроводной сети** на время игры.

## Идея

**GameTracker** — личный терминал игрока для взаимодействия с игрой. На полигоне все устройства работают в **сети LoRa**. **Организаторы** управляют игрой через **локальный сервер**. Трекер связан с **сервером** и взаимодействует с **объектами игры** через **NFC-метки** (предметы), **Wi-Fi** или **Bluetooth** (зоны, сценарное оборудование). Через **Wi-Fi** или **Bluetooth** можно настроить **трекер**, в том числе, используя **смартфон**. Кроме того, трекер позволяет определять местоположение игрока с помощью **спутниковой навигации**, а его ориентацию — с помощью **компаса**, **акселерометра** и **гироскопа**.

Информация выводится на **небольшой цветной дисплей**. Трекер также оснащён **вибромотором** и **динамиком**. **Логику** и **интерфейс** создают для каждого типа игры в **редакторах** для **ПК** и **смартфонов** — с **симуляцией** работы устройства и без знаний языков программирования.

## Инфраструктура


| Группа                     | Состав                                                              | Назначение                                                                     |
| -------------------------- | ------------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| **Устройства игрока**      | Игровой трекер                                                      | Персональный терминал: роль, статус, ввод действий, связь с сетью              |
| **Сервер и ретрансляторы** | Игровой сервер, LoRa-ретрансляторы                                  | Управление игрой, связь с трекерами и организаторами; расширение покрытия сети |
| **Инфраструктура поля**    | Оборудование для сценариев, аномалии, беспроводные поля или барьеры | Логика миссий, триггеры; игровой эффект в области, реакция при входе игрока    |
| **Интерактивные предметы** | Радиосвязь, метки (NFC/RFID)                                        | Взаимодействие на расстоянии или при контакте/близости                         |


## Интерактивные предметы


| Объект   | Протокол | Назначение                                            |
| -------- | -------- | ----------------------------------------------------- |
| Квест    | NFC      | Сдача или получение квеста, считывание сюжетных меток |
| Предмет  | NFC      | Использование или подбор игрового предмета            |
| Событие  | BLE      | Срабатывание события по близости                      |
| Аномалия | BLE      | Опасная зона, радиус устанавливается силой сигнала    |
| Мишень   | BLE      | Интерактивная мишень                                  |
| Оружие   | BLE      | Связь с электроникой специального игрового оружия     |
